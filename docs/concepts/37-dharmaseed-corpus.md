# 37 — Dharmaseed corpus (oral dharma talks)

> Status: **реализовано** — pilot Rob Burbea (458 talks, 16594 children).
>
> **Финальные решения после review:**
> 1. Variant A (single `dharma_v2` Qdrant collection, payload tag).
> 2. Deterministic context prefix (no LLM, $0 cost).
> 3. Full Burbea pilot (461 файлов → 458 inserted, 3 skipped без id-format).
> 4. Local-only — public consent campaign отложен (B-001-class).
> 5. Tradition: `pragmatic_dharma` (используем существующий, не создаём `modern-insight`).

## Что это

Dharmaseed.org — крупнейшая публичная библиотека **аудио-лекций современных
западных буддийских учителей** (~46 200 talks от ~418 teachers, 1974-2026).
Параллельно с Dharma-RAG идёт отдельный проект транскрипции: на `Q:\dharmaseed`
лежат **~34 878 готовых текстовых транскриптов от 79 учителей**, сделанных через
WhisperAI Pro (Whisper large-v3, browser-automation pipeline).

Этот концепт описывает, как впихнуть эти транскрипты в Dharma-RAG корпус:
схема, чанкинг, лицензии, чем отличается от каноники, какой пилот.

## Зачем у нас

**Yoniso flagship feature** (см. memory `project_dharma_rag_value_prop.md`).
Дhammaseed-транскрипты — это:

1. **Никем не использованы для обучения LLM** — Claude/GPT никогда не «слышали»
   Rob Burbea про soulmaking или Tara Brach про RAIN. Это значит, что любой
   ответ из этого корпуса даёт **уникальную ценность** vs ChatGPT-подписки.
2. **Современный язык** — не «mendicants, the four kinds...», а
   «when you sit down on the cushion and notice...». Доступно начинающим.
3. **Multi-teacher diversity** — для «бесконечного треда» (concept 36) сейчас
   95% корпуса — Sujato, диверсифицировать нечем. С Dharmaseed thread
   реально начинает крутиться: round 1 канон, round 2 Burbea, round 3 Tara
   Brach.
4. **Live-retreat и аудио-features** в будущем — талки имеют timestamps
   (если перетранскрибируем с faster-whisper), играть аудио из карточки,
   playlist по теме.

## Как работает

### 4.1. Что есть на Q-диске сейчас

```
Q:\dharmaseed\
├── transcripts\
│   ├── ajahn_sucitto\       # 2,969 .txt файлов
│   ├── rob_burbea\          # ~150 .txt
│   ├── tara_brach\          # ...
│   └── ... 79 teachers total
├── ajahn_sucitto_talks.json # metadata: id, title, date, mp3_url, name
├── rob_burbea_talks.json
└── ... 87 talks.json files
```

Формат одного транскрипта (`as_10231_opening_the_door.txt`):

```
WhisperAI Transcription Report
Title: as_10231_opening_the_door
Date: 4/17/2026
Duration: 1003 seconds
Language: auto

---

TRANSCRIPTION:

So to make a beginning, make an effort with the form, retreat form...
[~5-9K слов continuous prose, без таймстампов]

---

Generated by WhisperAI
```

То есть: **текст без timestamps**, заголовок + footer надо стрипать,
metadata уже отдельно лежит в `*_talks.json` (включая `mp3_url`,
`date`, original `title`).

### 4.2. Маппинг на FRBR (минимальные изменения)

FRBR-модель уже есть в `src/db/models/frbr.py` (концепт 02):
`Work` ←─ `Expression` ←─ `Instance` ←─ `Chunk`. Для talks применяем
**та же иерархия**, но с другим смыслом:

| FRBR-уровень | Sutta (sn56.11) | Dharma talk (rb_12345) |
|---|---|---|
| **Work** | Каноническая sutta — «Seting in Motion the Wheel of tthe Dhamma» | **Один talk** как event — «Soulmaking & Eros» (Rob Burbea, 2018-04-15) |
| **Expression** | Перевод (Sujato 2018, Bodhi 1995) | Транскрипция конкретным движком (Whisper large-v3 от 2026-04-17) |
| **Instance** | Конкретный snapshot HTML с SuttaCentral | Конкретный run транскрипции |
| **Chunk** | Parent (~1024 ток) + child (~384) | Так же, но без `segment_id` |

Это значит:
* **Каждый talk = новый Work** в `work_t`. canonical_id вида
  `rb_12345`, `as_10231` (teacher_slug + dharmaseed_id). Ожидаем
  ~35K новых строк в `work_t` (+ из них Phase 1 у нас всего ~1500).
* **Один Expression на talk** (одна транскрипция Whisper large-v3).
  Если retranscribe с faster-whisper — это новый Expression.
* **Author = teacher** через существующее `author_t` (есть поддержка
  `author_type='teacher'`).
* **Tradition** — нужна новая запись в `tradition_t` (например
  `'modern-insight'` для Vipassana lineage). Для теравадских монахов
  (Ajahn Sucitto, Bhante Bodhi) — `'theravada'`.
* **`segment_id` остаётся NULL** — у нас нет sub-talk идентификатора
  как у sutt («mn10:8.1»). Citation формат меняется на
  `rb_12345 · ¶3` (paragraph index). С timestamps (когда retranscribe)
  — `rb_12345 · 03:42`.

### 4.3. Чанкинг для устной речи

Канонический chunker (rag-day-07) опирается на segments. Здесь
segments нет → нужен **structural chunker для прозы**:

| Параметр | Канон | Talks |
|---|---|---|
| Parent token target | 1024-2048 | **1024-1500** (~5-8 минут речи) |
| Child token target | 256-384 | **256-384** (~1.5-2 минуты речи) |
| Boundary | segment | **paragraph break** (двойной newline в Whisper output) или `langchain.RecursiveCharacterTextSplitter` |
| Overlap (child) | 0 | **20-30%** child overlap для smooth retrieval |

Pre-готовые либы: `langchain_text_splitters.RecursiveCharacterTextSplitter`
с приоритетом `["\n\n", ". ", " "]` — стандартное решение для prose.

### 4.4. Contextual prefix

Тот же механизм rag-day-16 — Haiku 3.5 (или DeepSeek V4 после rag-day-35
fix) генерирует ~1-3 предложения «about-this-passage» префикс per child.
Промпт менять: вместо «sutta from Pāli Canon» — «excerpt from a recorded
dharma talk by {teacher_name} on {date}, titled {title}». Это **усиливает
retrieval** (BGE-M3 видит teacher/topic в embedding'е).

### 4.5. Пайплайн ingest'а

```
*.txt + *_talks.json
   │
   ▼
[strip header/footer + parse metadata]
   │
   ▼
[FRBR insert: Work + Expression + Instance + Chunk]
   │
   ▼
[contextualize_corpus.py — Haiku/DeepSeek prefix per child]
   │
   ▼
[reindex_qdrant_v2.py — BGE-M3 fp16 на GPU]
   │
   ▼
готов к retrieval
```

**Никаких изменений в `src/retrieval/`** — переиспользуем hybrid_search
прозрачно. BGE-M3 multilingual + sparse + dense обрабатывает английскую
ораторскую прозу не хуже sutt.

## Альтернативы

| Вариант | Почему не сейчас |
|---|---|
| **Не делать вовсе, остаться на каноне** | Тогда Yoniso = просто красивый sutta-search. Differentiation теряется. |
| **Использовать готовое RAG-решение от dharmaseed.org** | Они не предоставляют. Их сайт — audio-only listing. |
| **Перетранскрибировать всё с timestamps на faster-whisper локально** | ~35K talks × 30 мин = ~17,500 часов аудио. На 1080 Ti × 1.5x realtime = ~580 GPU-часов = **~24 дня non-stop**. Делается отдельно как Phase 5 enhancement. |
| **Только метаданные + ссылка на dharmaseed.org** | Тогда нет retrieval по содержанию, только по `title` — бессмысленно. |
| **Embed в OpenAI ada-002 / text-embedding-3** | Платно, vendor-lock, проигрывает BGE-M3 на multilingual + sparse. У нас уже работающий BGE-M3 stack. |
| **Один Work на teacher + Expression per talk** | Сэкономили бы JOIN-ы, но потеряли возможность citation на конкретный talk. Каждый talk — independent event, должен иметь свой Work. |

## Лицензии и consent — ХАРД constraint

Это самый важный нюанс. **Dharmaseed default license — CC BY-NC-ND 3.0/4.0**:
* BY — атрибуция teacher'а ✓ (мы можем)
* NC — non-commercial ✓ (Yoniso open-source)
* **ND — no-derivatives ✗** (RAG = derivative work, формально нарушение)

Плюс часть учителей имеет **более строгие персональные ограничения** —
Bhikkhu Bodhi, Joseph Goldstein и другие иногда требуют explicit permission.

**Stance проекта (предложенный):**

1. **Для local dev / experiments — ingest'им** на свою машину для
   валидации pipeline. Не публикуем ни `dharma_v2` Qdrant snapshot, ни
   индексы, ни derivative artifacts.
2. **Для public Yoniso (когда будет) — нужно явное разрешение** от:
   * Каждого включаемого teacher'а
   * Sangha Live (Rob Burbea estate, его материалы там)
   * Dharmaseed.org administration
   * Это **B-001-уровневый блокер** — outreach campaign по аналогии с
     buddhologist'ом, отдельный rag-day или separate non-code task.
3. **Документ `docs/CONSENT_LEDGER.md`** — таблица teacher / scope /
   date / permission_url. Без отметки в ledger — talk не попадает
   в публичный artifact.
4. **`expression_t.consent_ledger_ref`** уже есть в schema — заполняем
   при ingest'е.

> **Вывод:** ingest безопасен для **личного использования и dev'а**;
> public deploy ждёт consent campaign. На пилоте просто отметим в
> ledger «личный dev, не для публикации».

4. Run pilot: ingest Burbea ~150 talks → ~5-7K chunks → contextualize
   ($5) → encode (~5 мин GPU) → reindex.
5. Smoke retrieval: запросы вроде «What is soulmaking?», «Что такое
   восприятие в имажинале?» (для multilingual sanity), thread-mode
   на /thread по этим запросам.
6. **Если работает** — concept-doc 38 (multi-teacher scaling) +
   план для следующих teacher'ов.

**Что не делаем в пилоте:**

## Риски

1. **Quality дрейф golden set'а.** Сейчас golden = только канонические
   queries. После Burbea ingest'а top-5 на «What is satipatthana?»
   может включить Burbea-talk'и, которые НЕ в golden expected_works.
   Возможен false-positive regression. **Mitigation:** держать
   отдельный «teacher-mode» eval (новые queries про teacher'а),
   канонический golden оценивать на pre-Burbea config через flag.

2. **Stripping artefacts.** WhisperAI добавляет «WhisperAI Transcription
   Report» header и «Generated by WhisperAI» footer. Если стрипанём
   неаккуратно — embedding ловит boilerplate, retrieval mixes на
   queries вроде «whisper ai». **Mitigation:** строгий парсер с
   unit-test'ами на 5-10 sample-файлов.

3. **Whisper hallucinations.** Whisper large-v3 на длинных аудио
   иногда выдает повторы / hallucinated text (см. memory
   `feedback_validate_pipeline_outputs.md`). На pilot стадии валидируем
   вручную 5-10 talks. Систематический dedup pericope (rag-day-плана
   day-99-101) остаётся для Phase 4.

4. **Chunk-size mismatch с каноном.** Talk chunks 1024/384 — параметры
   тестировались на канонической прозе. Ораторская речь имеет другую
   density per token (более водянистая, повторы); возможно нужен
   tweaks. **Mitigation:** A/B на пилоте — посмотреть retrieval
   precision/recall по 10 ручным queries.

5. **Tradition / lineage classification spec.** Куда вешать teacher'а
   с mixed lineage (например Tara Brach — Vipassana + non-dual)? **Mitigation:**
   на пилоте Burbea = `tradition_code='modern-insight'`, дальше расширим.
